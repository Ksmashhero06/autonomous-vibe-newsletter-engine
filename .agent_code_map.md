# Project Architecture Map

## Root Directory

### server.ts
Express API backend. Handles:
- `/api/status`: Returns server key status and general run metrics/statistics.
- `/api/generate`: Runs the multi-agent generation pipeline. Exposes multiple live RSS scraper tools (HackerNews, TechCrunch, Google Blog, OpenAI News, Zoho Blog, Meta Research, Netflix TechBlog, AWS News Blog) to Agent A and the past issues memory tool to Agent B.
- `evaluateDraftSecurityAndQuality`: Programmatic security & quality guardrails checking for prompt injection, unclosed code blocks, and empty code blocks.
- 3-attempt feedback rewrite loop for Agent B if violations are detected.
- Serves Vite frontend in development and production modes.

### agent_pipeline.py
Python command-line multi-agent newsletter generation pipeline.
- Orchestrates:
  - Agent A (Trend Scout): Uses HackerNews, TechCrunch, Google Blog, OpenAI News, Zoho Blog, Meta Research, Netflix Tech, and AWS News RSS tools to source topics. If a custom topic is supplied, it deconstructs the topic into a structured sub-topic breakdown.
  - Agent B (Writer): Declares `check_past_issues` to read past issue history, autonomously filters out duplicates, and drafts a newsletter in Markdown. Supports self-correction loop when feedback is provided.
  - evaluate_draft_security_and_quality: Programmatic check for prompt injections, unclosed code blocks, and empty code blocks.
  - Agent C (Evaluator): Performs final quality checklist check on the approved draft.
- Supports optional custom topic research via `--topic` CLI flag.
- Saves generated newsletters to the `newsletters/` subdirectory.

### newsletters/
Directory storing all generated newsletter Markdown drafts.

### past_issues.json
Local JSON database representing the persistent newsletter memory database of covered topics. Used by `check_past_issues` to enforce content variation.

### agent_interactions.json
Conversational log representing direct messages and cooperation between the agents (Scout, Writer, Memory, Guardrails, Evaluator) during the last run.

### run_history.json
Day 5 execution telemetry database. Stores the last 50 pipeline run records, each containing:
- Timestamp, niche, model, status (success/failed)
- Agent A: last wake time, headlines pulled, RSS source
- Agent B: prompt_tokens, output_tokens, total_tokens, attempts count, guardrail violations list
- Agent C: compliance score (0-100), notes, passed verdict

### background_worker.py
Day 5 background fleet scheduler. Uses Python `time` + `threading` (no extra deps):
- Runs the pipeline on a configurable interval (default 5 min, supports decimals)
- Writes `background_worker_status.json` for Streamlit dashboard to poll
- Supports `--niche`, `--model`, `--interval`, `--simulate` CLI args
- Catches all pipeline exceptions and records failures to run_history.json
- Gracefully shuts down on Ctrl+C

### dashboard.py
Day 5 local observability Streamlit dashboard. Provides:
- Sidebar controls: niche, model, Custom Subject Topic (Optional), simulate mode, force manual run
- Background worker status indicator (idle / generating / stopped)
- 4 top-level metric cards: Last wake time, Total cycles, Accumulated tokens, Compliance rate
- **Tab 1** — Live Agent Cooperation: Real-time conversational timeline showing slack-style agent-to-agent cooperation messages (e.g. Writer requesting Memory check, Guardrail rejecting, Writer rewriting, Evaluator approving).
- **Tab 2** — Fleet Transaction Logs: Expandable entries per run with Agent A headlines, Agent B token stats, guardrail violations list, Agent C evaluator verdict
- **Tab 3** — Newsletter Archive: Dropdown file selector, full markdown rendering of any saved draft
- **Tab 4** — Token & Quality Metrics: Line charts for cumulative token consumption and evaluator score trends over time
