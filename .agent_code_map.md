# Project Architecture Map

## Root Directory

### server.ts
Express API backend. Handles:
- `/api/status`: Returns server key status and general run metrics/statistics.
- `/api/generate`: Runs the multi-agent generation pipeline. Exposes multiple live RSS scraper tools (HackerNews, TechCrunch, Google Blog, OpenAI News, Zoho Blog, Meta Research, Netflix TechBlog, AWS News Blog) to Agent A and the past issues memory tool to Agent B.
- Serves Vite frontend in development and production modes.

### agent_pipeline.py
Python command-line multi-agent newsletter generation pipeline.
- Orchestrates:
  - Agent A (Trend Scout): Uses HackerNews, TechCrunch, Google Blog, OpenAI News, Zoho Blog, Meta Research, Netflix Tech, and AWS News RSS tools to source topics.
  - Agent B (Writer): Declares `check_past_issues` to read past issue history, autonomously filters out duplicates, and drafts a newsletter in Markdown.
  - Agent C (Evaluator): Performs quality check on the draft.
- Saves generated newsletters to local markdown files.

### past_issues.json
Local JSON database representing the persistent newsletter memory database of covered topics. Used by `check_past_issues` to enforce content variation.

---

## .agents/

Workspace-level agent skills, rules, and workflows.

### .agents/skills/newsletter-generator/
Skill configuration and script execution for running the pipeline.
- `SKILL.md`: Metadata and guidelines for executing the pipeline.
- `scripts/agent_pipeline.py`: A copy of `agent_pipeline.py` used by the skill runner.

---

## src/

Vite React frontend.

### App.tsx
React dashboard UI. Handles:
- Setting parameters (niche, custom API key).
- Triggering pipeline generation (`/api/generate`).
- Parsing and displaying live agent log traces and final markdown output.
